# 行业模块：互联网、云计算、AI 与数据中心（Cloud, AI & Data Centers）

> 适用对象：超大规模云服务商、AI 基础设施企业、数据中心运营商、互联网平台、芯片设计公司
> 当前气候披露新热点——用电增长、水资源、Scope 3、碳移除争议集中

---

## 一、本行业最重要的气候风险与机遇

### 转型风险
- 电力消耗快速增长：AI 训练/推理和数据中心扩张推动用电量加速增长，可能抵消清洁能源努力
- Scope 3 供应链排放：服务器/芯片/建筑材料的生产排放，特别是半导体制造的高碳强度
- 碳移除可信度：依赖碳移除（CDR）达成净零目标面临技术成熟度和成本不确定性
- 可再生能源可获得性：大规模 PPA 需求可能推高区域绿电价格或面临供给瓶颈

### 物理风险
- 水资源约束：数据中心冷却用水量大，缺水地区面临运营中断和社区冲突
- 极端天气：影响数据中心运营连续性和供电稳定性
- 高温影响冷却效率：环境温度升高增加冷却成本和能源消耗

### 气候机遇
- 客户赋能减排：云服务替代本地部署可提升能效，AI 应用于气候解决方案
- 清洁能源采购的规模效应：大规模 PPA 可推动区域可再生能源建设
- 碳移除技术投资：早期投资碳移除技术可能形成竞争优势

| 条款 ID | 条款 | 本行业触发原因 |

## 三、最关键的投资有用指标

| 指标 | 投资模型变量 | 投资者关注重点 |
|---|---|---|
| PUE（Power Usage Effectiveness） | 运营效率/COGS | 数据中心能源效率及改善趋势 |
| WUE（Water Usage Effectiveness） | 运营风险 | 用水强度及缺水地区数据中心的水风险 |
| 可再生能源采购结构 | 碳成本/合规 | PPA vs 绿证 vs 24/7 清洁电力的占比和质量 |
| 数据中心用电总量及增速 | COGS/资本需求 | 绝对用电量和年增速，与收入增长的脱钩程度 |
| Scope 3 按类别拆分 | 供应链碳暴露 | 服务器/芯片/建材各自的排放占比 |
| 碳移除采购组合 | 净零可信度 | DACCS/生物炭/增强风化/造林的金额、交付量和永久性 |

## 四、常见误判

| 误判模式 | 正确判定 | 适用 ID |
|---|---|---|
| 100% 可再生能源采购 = 24/7 零碳电力 | 年度匹配 100% 不等于每小时匹配；绿证购买不改变实际用电的碳排放 | ID-065 |
| 购买碳移除 = Scope 1/2/3 实际减排 | 碳移除是目标外补充，不替代自身排放的绝对下降 | ID-093 |
| 披露 PUE = 披露气候相关财务影响 | PUE 是运营效率指标，不等于 §29(b)-(d) 要求的金额和百分比 | ID-068/069 |
| "碳负排放"承诺 = 已实现碳负排放 | 检查实际减排量 vs 碳移除采购量的基年对比 | ID-088 |
| AI 用于气候解决方案 = 自身气候机遇财务量化 | AI 赋能减排是产品功能描述，需量化为收入/资产金额 | ID-070 |
| 数据中心选址考虑气候 = 物理风险量化 | 选址决策是过程描述，S2 要求的是受影响资产金额和百分比 | ID-069 |

## 五、推荐 best-practice 做法类型

| 做法类型 | 说明 | 典型来源 |
|---|---|---|
| 碳移除组合透明度 | 按技术类型披露碳移除采购的金额、交付量、永久性和质量评估 | Microsoft |
| 内部碳费制度 | 覆盖 Scope 1+2+3 的内部碳费，碳费收入用于减排和碳移除 | Microsoft |
| Scope 3 按类别+材料拆分 | 披露服务器/芯片/建材等各自占 Scope 3 的百分比 | Microsoft |
| 绿电来源四类拆分 | 区分自发绿电/PPA/绿电产品/绿证的占比 | 台达电子 |
| 24/7 清洁能源追踪 | 按小时匹配清洁能源，披露匹配率和地理覆盖 | 待补充 |
| 水资源按数据中心定位 | 按数据中心所在地的水压力等级披露用水量和风险 | 待补充 |
