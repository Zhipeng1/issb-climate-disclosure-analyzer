# 条款计数验证测试（Clause Count Test）

> 每次输出前自动运行——确认合规大表行数和统计数据一致。

---

## 测试项

### T1：总行数 = 96

合规大表中独立评分的行数必须等于 96（不含表头、分组标题、总领段）。

验证方法：统计所有 `ID-XXX` 行的数量。

通过条件：`count(ID-XXX) == 96`

### T2：ID 连续且完整

ID-001 至 ID-096 每个 ID 必须出现且仅出现一次。

验证方法：检查 ID 序列是否连续。

通过条件：`set(IDs) == {ID-001, ID-002, ..., ID-096}`

### T3：统计一致性

摘要中的 ●/◐/○/不适用 各项条数之和必须等于 96。

验证方法：`score_full + score_partial + score_none + score_na == 96`

通过条件：总和 = 96，且百分比计算正确（允许四舍五入误差 ±0.5%）

### T4：S2 索引不计入

"S2 索引""TCFD 索引"等报告可读性事项不作为 96 条中的一条，应出现在"报告可读性与可检索性加分项"中。

验证方法：检查 ID-001 至 ID-096 对应的条款编号，确认没有"S2 索引"或"TCFD 索引"。

通过条件：96 条中无索引条目

### T5：不适用条款标注

非金融企业：ID-067（§29(a)(vi)(2) 融资排放）应标为"不适用"。
非特定行业：如有行业不适用的条款，须在评分列标注"—"并在建议列说明原因。

验证方法：检查不适用条款的评分和说明。

通过条件：不适用条款有合理解释

### T6：禁止循环标杆

当被评估公司在 best_practices 标杆库中时，合规大表"差距/改进建议"列不得引用同公司案例。

验证方法：提取被评估公司名称，检查改进建议列是否引用了同名公司。

通过条件：无同公司引用
